House Information Collector Terminology

집 모아의 용어를 먼저 고정합니다.

DB, 패키지, Cobra 커맨드, API 응답이 같은 언어를 쓰도록 한글명, 영어명, 변수명, 설명을 하나의 사전으로 관리합니다.

핵심 정의: 건물 안의 각 호실 하나를 HIC에서는 “주택”으로 표현합니다.

Core Domain

건물, 호실, 주택을 분리합니다.

건물 Building / building

물리적인 건축물 또는 동입니다. 하나의 건물 안에 여러 주택이 들어갈 수 있습니다.

주택 Housing Unit / housing_unit

서비스의 핵심 개별 정보 단위입니다. 건물 안의 각 호실 하나를 주택으로 표현합니다.

호실 Unit Number / unit_no

주택을 식별하는 번호입니다. 예: 0404, 0201, 1007.

Recruitment Scope

수집 대상은 모집공고 계열만입니다.

포함 Recruitment Family

모집공고, 정정공고, 추가모집, 잔여세대 모집은 모두 모집공고 계열로 포함합니다.

제외 Non-Recruitment Post

당첨자 발표, 계약안내, 시스템 공지, 일반 알림은 아예 수집하지 않습니다.

Workflow Terms

각 레이어는 서로 다른 산출물을 만듭니다.

Discovery

모집공고와 첨부 메타를 발견합니다.

Extraction

PDF/XLSX/HWP/HTML에서 텍스트와 행을 추출합니다.

Normalize

주소, 주택, 가격, 일정, 상호전환을 필드화합니다.

LLM Assist

낮은 confidence 구간만 보정합니다.

QA Gate

필수 필드가 채워진 데이터만 서비스로 승격합니다.

Normalized Fields

Raw에 묻히면 안 되는 필드입니다.

한글 영어 축약어/변수명 설명
주소 Address address 주택의 전체 소재지 주소. 필수 정규화 필드입니다.
자치구 District district 서울시 구 단위 행정구역. 예: 영등포구.
주택구조 Structure Type structure_type 개방형원룸, 분리형원룸 등 구조 유형.
엘리베이터 설치 여부 Elevator Installed elevator_installed 설치/미설치를 boolean으로 정규화합니다.
임대보증금 Deposit deposit_krw 원 단위 숫자 필드로 저장합니다.
월임대료 Monthly Rent monthly_rent_krw 원 단위 숫자 필드로 저장합니다.

Rent Conversion

상호전환은 규칙과 계산값을 분리합니다.

전환 규칙 Rent Conversion Rule

최대 전환비율, 연 전환요율, 10,000원당 추가보증금, 전환 가능 여부를 공고 단위로 저장합니다.

주택별 계산값 Unit Rent Conversion Value

각 주택의 월임대료를 기준으로 최대 전환 가능 월임대료, 예상 추가보증금, 전환 후 최저 월임대료를 계산합니다.

Source Trace

정규화 값은 근거 위치를 가져야 합니다.

한글 영어 축약어/변수명 설명
원본 위치 Source Span source_span PDF 페이지, XLSX sheet/row/cell, HTML selector 등 필드의 근거 위치입니다.
소스 행 Source Row source_row XLSX 원본 행 JSON. 조회 기준이 아니라 감사/재처리용 보조 증거입니다.
신뢰도 Confidence confidence 필드 추출/정규화 결과의 신뢰도입니다.
