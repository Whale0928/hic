핵심 수정 내역

SH

  • source_span을 로컬 object-root 경로에 덜 의존하도록 정리해 재수집 idempotency를 개선했다.
  • 중복 pending/approved 충돌을 줄이도록 artifact/offering upsert와 QA 승격 흐름을 보강했다.
  • HWP 본문 추출 fallback을 추가해 마곡 도전숙 공고를 정규화했다.
  • 두레주택/희망하우징/든든주택처럼 호실 단위가 아닌 공급호수형 신청 단위를 Offering으로 정리했다.
  • SH 활성 청약 목록을 함께 조회해 게시판 페이징 바깥의 신청 가능 공고 누락을 보완했다.

LH

  • LH/MyHome API 수집을 SH와 독립된 경로로 추가했다.
  • 분양 납부금액(enty, prtpay, surlus)을 QA 가능한 금액 필드로 매핑했다.
  • sumSuplyCo=0인 전세임대는 suplyHoCo 문구에서 공급호수를 보정했다.
  • MyHome 상세 HTML에서 지원한도액, 임대보증금 조건, 월임대료 조건을 추출했다.
  • 상세 HTML이 부족한 매입임대는 공고문 PDF를 다운로드/보존/추출해 fallback 정규화했다.

운영 메모

  • 비모집 공고, 당첨자 발표, 계약 안내, 접수 결과는 normal archive에 저장하지 않는 정책을 유지한다.
  • Serving/API는 qa_status='approved' 조건을 기본 필터로 사용한다.
  • MyHome 상세/PDF fallback은 API 필드만으로 QA가 어려운 항목에 한정해 실행한다.
  • LLM 정제는 이번 최종 검증에는 사용하지 않았다. 향후 사용 시 사용자 제한대로 최대 1500회 안에서 운영하면 된다.
